## 1. AgentScope 框架概述

### 1.1 什么是 AgentScope？

AgentScope 是一个专为多智能体应用设计的 Python 框架，由阿里巴巴达摩院开发。它的核心特点：

- **消息驱动**：智能体之间通过消息（Msg）进行通信
- **易于扩展**：模块化设计，支持自定义智能体和工具
- **并发支持**：内置异步处理和并发管道
- **结构化输出**：集成 Pydantic 模型约束智能体行为

### 1.2 核心组件

```
AgentScope 框架
├── Agent（智能体）：执行任务的基本单元
├── Message（消息）：智能体间通信的载体
├── Model（模型）：底层 LLM 接口封装
├── Pipeline（管道）：智能体协作流程控制
└── MsgHub（消息中心）：多智能体消息路由
```

## 3. 核心概念详解

### 3.1 消息（Message）

消息是 AgentScope 中智能体通信的基本单位。

```python
from agentscope.message import Msg

# 创建消息
msg = Msg(
    name="游戏主持人",           # 发送者名称
    content="游戏开始！",        # 消息内容
    role="system",              # 角色类型：system/user/assistant
    metadata={"round": 1}       # 附加元数据（可选）
)
```

**关键点**：
- 所有智能体交互都通过 Msg 对象
- 支持附加元数据传递结构化信息
- 消息历史自动维护，便于上下文管理

### 3.2 智能体（Agent）

智能体是执行任务的基本单元，本项目使用 `ReActAgent`。

```python
from agentscope.agent import ReActAgent
from agentscope.model import DashScopeChatModel

agent = ReActAgent(
    name="刘备",                              # 智能体名称
    sys_prompt="你是刘备，扮演预言家...",     # 系统提示词
    model=DashScopeChatModel(                # 底层模型
        model_name="qwen-max",
        api_key=os.environ["DASHSCOPE_API_KEY"],
        enable_thinking=True,                # 启用思考链
    ),
)
```

**ReActAgent 特点**：
- 支持推理（Reasoning）和行动（Acting）循环
- 可以使用工具和函数调用
- 内置思考链（Chain of Thought）能力

### 3.3 消息中心（MsgHub）

MsgHub 是多智能体协作的核心组件，负责消息路由和广播。

```python
async with MsgHub(
    participants=self.werewolves,           # 参与者列表
    enable_auto_broadcast=True,             # 自动广播消息
    announcement=initial_msg                # 初始公告
) as hub:
    # 在这个上下文中，所有参与者的消息会自动广播给其他人
    for wolf in self.werewolves:
        await wolf(structured_model=DiscussionModelCN)
    
    # 关闭自动广播
    hub.set_auto_broadcast(False)
```

**使用场景**：
- 狼人夜晚讨论（只有狼人能看到彼此消息）
- 白天全员讨论（所有存活玩家互相可见）
- 私密投票（关闭广播，收集个人决策）


### 3.4 管道（Pipeline）

管道用于控制多个智能体的执行流程。

#### 顺序管道（Sequential Pipeline）
```python
# 玩家依次发言
await sequential_pipeline(self.alive_players)
```
- 智能体按顺序执行
- 每个智能体可以看到前面智能体的输出
- 适用于：轮流发言、依次决策

#### 扇出管道（Fanout Pipeline）
```python
# 所有玩家同时投票
vote_msgs = await fanout_pipeline(
    self.alive_players,                          # 参与者
    msg=await self.moderator.announce("请投票"), # 触发消息
    structured_model=get_vote_model_cn(...),     # 输出格式
    enable_gather=False,                         # 不汇总结果
)
```
- 所有智能体并发执行
- 提高效率，适合独立决策
- 适用于：投票、同时行动

### 3.5 结构化输出（Structured Output）

使用 Pydantic 模型约束智能体输出格式。

```python
from pydantic import BaseModel, Field

class VoteModelCN(BaseModel):
    """投票输出格式"""
    vote: str = Field(description="投票目标玩家姓名")
    reason: str = Field(description="投票理由")
    suspicion_level: int = Field(ge=1, le=10, description="怀疑程度")
```

**优势**：
- 确保输出格式一致
- 自动验证数据类型
- 便于后续处理和统计
- 提高系统稳定性

**使用方式**：
```python
# 调用智能体时传入结构化模型
response = await agent(structured_model=VoteModelCN)

# 访问结构化数据
target = response.metadata.get("vote")
reason = response.metadata.get("reason")
```

### 4.3 structured_output_cn.py - 结构化输出模块

```python
class DiscussionModelCN(BaseModel):
    """讨论输出格式"""
    reach_agreement: bool = Field(description="是否已达成一致意见")
    confidence_level: int = Field(description="信心程度(1-10)", ge=1, le=10)
    key_evidence: Optional[str] = Field(description="关键证据", default=None)
```

#### 动态投票模型
```python
def get_vote_model_cn(agents: list[AgentBase]) -> type[BaseModel]:
    """根据当前存活玩家动态生成投票模型"""
    class VoteModelCN(BaseModel):
        vote: Literal[tuple(_.name for _ in agents)] = Field(
            description="你要投票淘汰的玩家姓名"
        )
        reason: str = Field(description="投票理由")
        suspicion_level: int = Field(description="怀疑程度", ge=1, le=10)
    
    return VoteModelCN
```

**技术亮点**：
- 使用 `Literal` 类型限制投票目标只能是存活玩家

#### 投票统计
```python
def majority_vote_cn(votes: Dict[str, str]) -> tuple[str, int]:
    """多数投票统计"""
    if not votes:
        return "无人", 0
    
    vote_counts = Counter(votes.values())
    most_voted = vote_counts.most_common(1)[0]
    
    return most_voted[0], most_voted[1]
```

#### 胜负判定
```python
def check_winning_cn(alive_players: List[AgentBase], 
                     roles: Dict[str, str]) -> Optional[str]:
    """检查游戏胜利条件"""
    alive_roles = [roles.get(p.name, "村民") for p in alive_players]
    werewolf_count = alive_roles.count("狼人")
    villager_count = len(alive_roles) - werewolf_count
    
    if werewolf_count == 0:
        return "好人阵营胜利！所有狼人已被淘汰！"
    elif werewolf_count >= villager_count:
        return "狼人阵营胜利！狼人数量已达到或超过好人！"
    
    return None
```

**2. 狼人阶段**
```python
async def werewolf_phase(self, round_num: int):
    """狼人阶段"""
    if not self.werewolves:
        return None
    
    # 使用 MsgHub 创建狼人私密讨论空间
    async with MsgHub(
        self.werewolves,
        enable_auto_broadcast=True,
        announcement=await self.moderator.announce("狼人们，请讨论今晚的击杀目标...")
    ) as werewolves_hub:
        # 讨论阶段
        for _ in range(MAX_DISCUSSION_ROUND):
            for wolf in self.werewolves:
                await wolf(structured_model=DiscussionModelCN)
        
        # 投票击杀
        werewolves_hub.set_auto_broadcast(False)
        kill_votes = await fanout_pipeline(
            self.werewolves,
            msg=await self.moderator.announce("请选择击杀目标"),
            structured_model=WerewolfKillModelCN,
            enable_gather=False,
        )
        
        # 统计投票
        votes = {}
        for i, vote_msg in enumerate(kill_votes):
            if vote_msg and hasattr(vote_msg, 'metadata') and vote_msg.metadata:
                votes[self.werewolves[i].name] = vote_msg.metadata.get("target")
        
        killed_player, _ = majority_vote_cn(votes)
        return killed_player
```

**流程说明**：
1. 创建 MsgHub，只有狼人参与
2. 多轮讨论，狼人互相交流
3. 关闭自动广播，进行私密投票
4. 使用 fanout_pipeline 并发收集投票
5. 统计投票结果，返回被击杀玩家


**3. 白天阶段**
```python
async def day_phase(self, round_num: int):
    """白天阶段"""
    await self.moderator.day_announcement(round_num)
    
    # 全员讨论
    async with MsgHub(
        self.alive_players,
        enable_auto_broadcast=True,
        announcement=await self.moderator.announce("现在开始自由讨论...")
    ) as all_hub:
        # 顺序发言
        await sequential_pipeline(self.alive_players)
        
        # 投票阶段
        all_hub.set_auto_broadcast(False)
        vote_msgs = await fanout_pipeline(
            self.alive_players,
            await self.moderator.announce("请投票选择要淘汰的玩家"),
            structured_model=get_vote_model_cn(self.alive_players),
            enable_gather=False,
        )
        
        # 统计投票
        votes = {}
        for i, vote_msg in enumerate(vote_msgs):
            if vote_msg and hasattr(vote_msg, 'metadata') and vote_msg.metadata:
                votes[self.alive_players[i].name] = vote_msg.metadata.get("vote")
        
        voted_out, vote_count = majority_vote_cn(votes)
        await self.moderator.vote_result_announcement(voted_out, vote_count)
        
        return voted_out
```

**与狼人阶段的区别**：
- 参与者是所有存活玩家（而非只有狼人）
- 使用 `sequential_pipeline` 顺序发言（而非并发讨论）
- 投票目标限制为存活玩家（通过动态生成的 VoteModel）

**4. 游戏主循环**
```python
async def run_game(self):
    """运行游戏主循环"""
    await self.setup_game()
    
    for round_num in range(1, MAX_GAME_ROUND + 1):
        # 夜晚阶段
        await self.moderator.night_announcement(round_num)
        killed_player = await self.werewolf_phase(round_num)
        await self.seer_phase()
        final_killed, poisoned_player = await self.witch_phase(killed_player)
        
        # 更新死亡玩家
        night_deaths = [p for p in [final_killed, poisoned_player] if p]
        self.update_alive_players(night_deaths)
        await self.moderator.death_announcement(night_deaths)
        
        # 检查胜利条件
        winner = check_winning_cn(self.alive_players, self.roles)
        if winner:
            await self.moderator.game_over_announcement(winner)
            return
        
        # 白天阶段
        voted_out = await self.day_phase(round_num)
        hunter_shot = await self.hunter_phase(voted_out)
        
        # 更新死亡玩家
        day_deaths = [p for p in [voted_out, hunter_shot] if p]
        self.update_alive_players(day_deaths)
        
        # 检查胜利条件
        winner = check_winning_cn(self.alive_players, self.roles)
        if winner:
            await self.moderator.game_over_announcement(winner)
            return
```

**游戏流程**：
```
初始化游戏
↓
循环开始（最多 MAX_GAME_ROUND 轮）
├── 夜晚阶段
│   ├── 狼人击杀
│   ├── 预言家查验
│   ├── 女巫行动
│   ├── 更新死亡玩家
│   └── 检查胜利条件
├── 白天阶段
│   ├── 全员讨论
│   ├── 投票淘汰
│   ├── 猎人技能
│   ├── 更新死亡玩家
│   └── 检查胜利条件
└── 循环结束或游戏结束
```

## 5. 关键技术点

### 5.1 异步编程（Async/Await）

整个项目使用 Python 的异步编程模式。

```python
# 异步函数定义
async def werewolf_phase(self, round_num: int):
    # 异步等待
    killed_player = await self.werewolf_phase(round_num)
    
    # 异步上下文管理器
    async with MsgHub(...) as hub:
        # 异步操作
        await wolf(structured_model=DiscussionModelCN)

# 运行异步主函数
asyncio.run(main())
```

**为什么使用异步？**
- 多个智能体可以并发执行，提高效率
- 等待 LLM 响应时不阻塞其他操作
- 更好地模拟真实的多人游戏场景

### 5.2 容错机制

代码中有多处容错处理，确保单个智能体异常不影响整体游戏。

```python
# 检查返回结果是否有效
if vote_msg is not None and hasattr(vote_msg, 'metadata') and vote_msg.metadata is not None:
    votes[self.alive_players[i].name] = vote_msg.metadata.get("vote")
else:
    # 如果返回无效，视为弃票
    print(f"⚠️ {self.alive_players[i].name} 的投票无效，视为弃票")
    votes[self.alive_players[i].name] = None
```

**容错策略**：
- 检查消息对象是否为 None
- 检查 metadata 属性是否存在
- 提供默认值或随机选择
- 记录警告日志但继续游戏

### 5.3 状态管理

游戏状态通过多个列表和字典维护。

### 5.5 提示词工程

提示词设计是影响智能体表现的关键因素。

#### 提示词结构
```
1. 身份设定：你是{character}，扮演{role}
2. 输出格式：严格按照JSON格式回复
3. 角色特点：列出角色能力和目标
4. 性格要求：以特定人物性格说话
```

#### 示例分析
```python
base_prompt = f"""你是{character}，在这场三国狼人杀游戏中扮演{role}。

请严格按照以下JSON格式回复，不要添加任何其他文字：
{{
    "reach_agreement": true/false,
    "confidence_level": 1-10的数字,
    "key_evidence": "你的证据或观点"
}}

角色特点：
- 你是狼人阵营，目标是消灭所有好人
- 夜晚可以与其他狼人协商击杀目标
- 白天要隐藏身份，误导好人
- 以{character}的性格说话和行动
"""
```

**设计要点**：
1. **明确身份**：同时告知游戏角色和人物性格
2. **格式约束**：在提示词中说明输出格式（配合结构化输出）
3. **目标导向**：清晰说明胜利条件和行动策略
4. **性格融入**：要求以特定人物风格表达

### 5.6 消息路由策略

不同阶段使用不同的消息路由策略。

| 阶段 | 参与者 | 广播模式 | 管道类型 | 说明 |
|------|--------|----------|----------|------|
| 狼人讨论 | 狼人 | 自动广播 | Sequential | 狼人互相可见 |
| 狼人投票 | 狼人 | 关闭广播 | Fanout | 私密投票 |
| 预言家查验 | 预言家 | 无 | 单独调用 | 私密行动 |
| 女巫行动 | 女巫 | 无 | 单独调用 | 私密行动 |
| 白天讨论 | 全员 | 自动广播 | Sequential | 全员可见 |
| 白天投票 | 全员 | 关闭广播 | Fanout | 公开投票 |

**关键代码**：
```python
# 开启自动广播 - 消息互相可见
async with MsgHub(participants, enable_auto_broadcast=True) as hub:
    await sequential_pipeline(participants)
    
    # 关闭自动广播 - 私密投票
    hub.set_auto_broadcast(False)
    votes = await fanout_pipeline(participants, ...)
```

## 7. 常见问题与解答

### Q1: 为什么使用 ReActAgent 而不是普通的 DialogAgent？

**A:** ReActAgent 支持推理（Reasoning）和行动（Acting）循环，更适合需要复杂决策的场景。在狼人杀中，智能体需要：
- 分析其他玩家的发言
- 推理谁可能是狼人
- 决定投票或行动目标

ReActAgent 的思考链能力可以让这个过程更加透明和可控。

### Q2: 结构化输出有时会失败怎么办？

**A:** 项目中已经实现了容错机制：
```python
if vote_msg and hasattr(vote_msg, 'metadata') and vote_msg.metadata:
    votes[player.name] = vote_msg.metadata.get("vote")
else:
    # 提供默认值
    votes[player.name] = None
```

**改进建议**：
- 在提示词中更明确地说明输出格式
- 使用更强大的模型（如 qwen-max）
- 添加重试机制
- 提供输出示例

### Q3: 如何调试智能体的行为？

**A:** 几种调试方法：

1. **查看消息历史**

2. **启用详细日志**

### Q4: 如何提升智能体的游戏水平？

**A:** 几个方向：

1. **优化提示词**
   - 提供更多游戏策略指导

### Q6: 能否让人类玩家参与？

**A:** 可以！需要创建一个 `HumanAgent` 类：
```python
class HumanAgent(AgentBase):
    async def __call__(self, msg=None, structured_model=None):
        # 显示消息给人类
        print(f"\n{msg.content if msg else '请做出决策'}")
        
        # 获取人类输入
        user_input = input("你的回复: ")
        
        # 返回消息
        return Msg(name=self.name, content=user_input, role="user")
```

然后在创建玩家时混合使用 AI 和人类玩家。

### Q7: 如何实现游戏回放？

**A:** 保存游戏状态和消息历史：
```python
class GameRecorder:
    def __init__(self):
        self.events = []
    
    def record_event(self, event_type, data):
        self.events.append({
            "timestamp": time.time(),
            "type": event_type,
            "data": data
        })
    
    def save_to_file(self, filename):
        with open(filename, 'w') as f:
            json.dump(self.events, f, indent=2)
```

在游戏的关键节点调用 `record_event`。

## 10. 总结

### 10.1 核心要点回顾

1. **消息驱动架构**
   - 智能体通过 Msg 对象通信
   - MsgHub 负责消息路由和广播
   - 支持异步并发处理

2. **结构化输出**
   - 使用 Pydantic 模型约束输出
   - 动态生成模型适应游戏状态
   - 提高系统稳定性和可预测性

3. **管道机制**
   - Sequential Pipeline: 顺序执行
   - Fanout Pipeline: 并发执行
   - 灵活组合实现复杂流程

4. **容错设计**
   - 检查返回值有效性
   - 提供默认值和降级策略
   - 单个智能体异常不影响整体

5. **双重身份建模**
   - 游戏角色 + 三国人物
   - 提示词融合角色能力和性格
   - 增强游戏趣味性和真实感
